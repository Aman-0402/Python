Question 1

Select the incorrect statement.

  • A) You can pass keyword arguments in any order.
  • B) You can call a function with positional and keyword arguments.

  • D) You can pass positional arguments in any order.

‚úÖ Correct Answer: You can pass positional arguments in any order. ‚ùå

üß† Explanation:

Let's look at each option:

‚ùå You can pass positional arguments in any order.
‚û§ Incorrect. The order of positional arguments matters because the function assigns them based on their position.

    print(f"{name} is {age} years old.")

‚û§ This is true because keyword arguments are explicitly named, so order doesn't matter.

‚úÖ You can call a function with positional and keyword arguments.
‚û§ Yes, a mix is allowed, but positional must come first.

‚úÖ Positional arguments must be before keyword arguments in a function call.
‚û§ This is correct and mandatory in Python syntax.

‚úÖ Final Answer: You can pass positional arguments in any order.

Question 5

Question 7

What is the order of precedence in Python?

  1. Parentheses
  2. Exponential
  3. Multiplication
  4. Division
  5. Addition
  6. Subtraction
  • A) i, ii, iii, iv, v, vi
  • B) ii, i, iii, iv, v, vi
  • C) ii, i, iv, iii, v, vi
  • D) i, ii, iii, iv, vi, v

‚úÖ Correct Answer: i, ii, iii, iv, v, vi

üß† Explanation: Order of Precedence in Python (from highest to lowest):

  1. Parentheses ‚Üí ()
    ‚û§ Used to override default precedence
  2. Exponential ‚Üí **
  3. Multiplication ‚Üí *
  4. Division ‚Üí /, //, %
  5. Addition ‚Üí +
  6. Subtraction ‚Üí -

Example:

result = (3 + 2) ** 2 * 4 / 2 - 1
# Steps:
1. (3+2) = 5    (Parentheses first)
2. 5**2 = 25    (Exponentiation)
3. 25*4 = 100   (Multiplication)
4. 100/2 = 50   (Division)
5. 50-1 = 49    (Subtraction last)

Key Points:

  • Multiplication and Division have same precedence (left to right)
  • Addition and Subtraction have same precedence (left to right)
  • Use parentheses to explicitly define evaluation order

Question 8

What is the correct order of precedence in Python?

  1. Parentheses
  2. Exponentiation (**)
  3. Multiplication / Division
  4. Addition / Subtraction
  • A) i, ii, iii, iv
  • B) ii, i, iii, iv
  • C) i, iii, ii, iv
  • D) i, ii, iv, iii

‚úÖ Correct Answer: A. i, ii, iii, iv

üß† Explanation:

Python follows this order of precedence for arithmetic operators:

  1. Parentheses () ‚Äì evaluated first
  2. Exponentiation **
  3. Multiplication * / Division / / Floor division // / Modulus %
  4. Addition + / Subtraction -

So: i ‚Üí ii ‚Üí iii ‚Üí iv is the correct sequence.

Practical Example:

result = (5 + 3) ** 2 * 4 / 2 - 1
# Evaluation steps:
1. (5 + 3) = 8       (Parentheses first)
2. 8 ** 2 = 64       (Exponentiation)
3. 64 * 4 = 256      (Multiplication)
4. 256 / 2 = 128     (Division)
5. 128 - 1 = 127     (Subtraction last)

Key Notes:

  • Operators with same precedence (like * and /) evaluate left-to-right
  • Parentheses always have highest precedence
  • Exponentiation comes before other arithmetic operations
  • Use parentheses to make complex expressions clearer

Question 13

Which function overloads the >> operator?

  • A) more()
  • B) gt()
  • C) ge()
  • D) None of the above

‚úÖ Correct Answer: D. None of the above

üß† Explanation:

In Python, operator overloading is done by defining special methods (also known as magic methods or dunder methods).

The >> (right shift) operator is overloaded using:

__rshift__(self, other)

Analysis of Options:

Option Evaluation Correct Method
A) more() ‚ùå Not a special method Doesn't exist
B) gt() ‚ùå Incorrect name __gt__() (for > operator)
C) ge() ‚ùå Incorrect name __ge__() (for >= operator)
D) None ‚úÖ Correct The right method is __rshift__()

Common Operator Overloading Methods:

  • __add__() for + operator
  • __sub__() for - operator
  • __mul__() for * operator
  • __rshift__() for >> operator
  • __lshift__() for << operator

Example Implementation:

class Data:
    def __rshift__(self, other):
        print(f"Right shift with {other}")

d = Data()
d >> 5  # Output: Right shift with 5

Question 14

  • A) __add__()

  • C) __sum__()
  • D) __concat__()

‚úÖ Correct Answer: A. __add__()

üß† Explanation:

Python supports operator overloading using special methods (also called dunder methods).

For the + operator, the method is:

Comparison with Original Question:

For the >> operator (from previous question), the method is __rshift__(), making "None of the above" the correct answer there.

Common Operator Overloading Methods:

Operator Method Example
+ __add__() a + b
- __sub__() a - b
* __mul__() a * b
>> __rshift__() a >> b

Example Implementation:

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

v1 = Vector(1, 2)
v2 = Vector(3, 4)
v3 = v1 + v2  # Uses __add__ method

Key Points:

  • Special methods always have double underscores (dunder)
  • __add__() is called when + operator is used
  • Can return any type (not limited to same class)
  • Should implement __radd__() for reverse operations

  • Not to be confused with logical OR (which uses or keyword)

Operator Method Purpose

// __floordiv__() Floor division

    def __init__(self, value):
        self.value = value
        
    def __or__(self, other):

f1 = Flags(0b01)
f2 = Flags(0b10)

print(bin(result.value))  # Output: 0b11

Common Uses:

  • Bitwise operations
  • Combining sets or flags
  • Custom OR-like behavior for objects

Question 19

Option Syntax Key Difference

  • Arrays require importing a module (import array)
